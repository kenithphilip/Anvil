🗄️ Schema Upgrades (optional)

    
Run these SQL commands in your Supabase SQL Editor to enable advanced features. Each is optional — the app degrades gracefully if columns are missing.


    

      

        Source-order & hierarchy preservation
        
      

      
Adds seq_no (BOM source-file row order) and level (assembly hierarchy depth) so the gun BOM page displays parts in the original Excel order with assembly/sub-assembly/part visual indicators.

      
ALTER TABLE bom_items ADD COLUMN IF NOT EXISTS seq_no INTEGER;
ALTER TABLE bom_items ADD COLUMN IF NOT EXISTS level INTEGER;

    


    

      

        Fractional quantities
        
      

      
Upgrades qty from INTEGER to NUMERIC so China BOM material entries (cable lengths like 2.4m, glue weight 0.01kg) import without rounding.

      
ALTER TABLE bom_items ALTER COLUMN qty TYPE NUMERIC USING qty::NUMERIC;

📥 Import BOM

  
Upload Bill of Materials files from any Obara source. The format is auto-detected and metadata extracted before import.


  

    ℹ️ How this works — formats supported, what to expect
    

      
Supported sources: 🇮🇳 Obara India · 🇰🇷 Obara Korea · 🇨🇳 Obara China · 🇯🇵 Obara Japan. Each has its own column layout, hierarchy markers, and metadata block — all auto-detected from header row signatures, language characters, and identifying labels.

      
Modification detection: if you upload a BOM for a gun number that already exists, the importer fetches the current BOM and shows a diff (parts added, removed, changed) so you can review before replacing.

      
Hierarchy & order: source-file row order (seq_no) and assembly depth (level) are captured so you see parts in the original Excel sequence with L1/L2/L3 visual indicators on the gun page.

      
Gun number: auto-suggested from the format's metadata (PRODUCT CODE for China, Item No. for Japan, Drawing No. for Korea, filename for India). Always editable in the file row before import.

⚙️ Configure Spare Columns

    
  

  
Select spare types to add as columns in your matrix. Already-existing columns are checked and locked.


  

    ℹ️ How auto-fill matches parts to columns
    

      
Path A — Part name keyword: A column named BUSH matches any part whose name starts with BUSH. For Chinese-prefixed names (枪体主件Main Body Assy) the matcher strips the leading non-Latin run before testing — so a column named Main Body still matches.

      
Path B — Part-number pattern: Some columns have built-in regex patterns to catch parts by part-number convention even when the name is in another language or doesn't contain the keyword. Built-in patterns:

      

        
TIP CAP — T-16-D, T13D, TC-16-A

        
O-RING — OR-12, P-24, ORG-12

        
BACKING PLATE — BG-2C16

        
BOLT — GB/T 70-85, M8*25, HCS

        
WASHER — GB/T 93-85, GB/T 95-85, WSH

        
NUT — GB/T 6170

        
PIN — GB/T 119, PN-12

        
BEARING — B-2007, BRG-, GB/T 276

        
ELECTRODE — E-12, ELEC-

        
… plus OIL SEAL, BUSH, KEY, CIRCLIP, TIP

      

      
Path C — Consumable filter: Columns marked Consumable additionally require copper-type material (CuCrZr, BE14C, C1100, etc.) so non-conducting parts named "ADAPTER" or "HOLDER" don't slip in.
